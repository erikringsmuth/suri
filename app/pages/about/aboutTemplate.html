How websites get data

When you opened this page the browser made a GET {{url}} HTTP/1.1 request to get the page from the server. Every time you go to a new page the browser does a GET request to load the page. GET tells the browser to read data. There are also create POST, update PUT, and delete DELETE methods to change data. A static web page like this will only let you GET data.

Types of data

A website is made up of HTML, CSS, and JavaScript. You will only be able to GET this type of data. On a sites like Twitter, Facebook, and suri.io you send JSON to and from the server to update your status, change your profile, etc. This is done using JavaScript and is called an AJAX request or XHR. When you create a new status you POST data to the server. When you update your profile you PUT data to the server. Suri was made to let you play around with these APIs and let you see how the raw data is sent and received.

Security

Signing into suri.io uses Google's OAuth2 and OpenID Connect over HTTPS. Using OAuth means suri never needs to know your username or password. When you authorize access with Google it will send an id_token back to suri. Suri uses your ID to determine who you are and what you can do.

When you use Suri, non-CORS enabled APIs have to send the request through the suri.io server to avoid XSS. Browsers don't allow calling hosts other than the current host ({{hostname}}) for security reasons. CORS enabled APIs will directly call the endpoint from your browser. These APIs don't need to be proxied through suri.io.

Terms

  • API - Application programming interface. This defines what the data looks like and how you interact with it.
  • AJAX - Asynchronous JavaScript and XML. When JavaScript makes a call to the server it's called an AJAX request.
  • XHR - XML HTTP Request. Another name for AJAX.
  • HTTP - Hypertext Transfer Protocol. Most web data is sent using HTTP and HTTPS (secure). See how the current URL {{url}} starts with HTTP? That means it got the web page using the HTTP protocol.
  • REST - Representational state transfer. An architectural API style that defines a consistent way of interacting with data.
  • CORS - Cross-Origin Resource Sharing. The ability to make an AJAX call to a different host (other than {{hostname}} for example).
  • XSS - Cross-site scripting. This is blocked by the browser. XSS would allow the JavaScript to send and receive data from a host other than {{hostname}}.
  • JSON - JavaScript Object Notation. The JavaScript data format.
  • XML - Extensible Markup Language. Another data format popular in older systems.
  • Host - The server providing the data.
  • Proxy - An intermediate server that data is sent through before going to the destination server.
  • HTML - HyperText Markup Language. The language for creating web pages.
  • CSS - Cascading Style Sheets. Defines the look and formatting of the document.
  • JavaScript - The programming language used in web browsers.
